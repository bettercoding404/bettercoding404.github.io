## 基础概念
打印格式化本质上是一种控制文本输出外观的机制。在 Python 中，我们可以使用不同的方法来格式化字符串，以便在调用 `print()` 函数时，输出的内容按照我们期望的格式呈现。例如，我们可能希望将数字格式化为带有特定小数位数的字符串，或者将文本对齐到特定的宽度。

## 使用方法

### 旧风格格式化
旧风格格式化使用 `%` 操作符来格式化字符串。它的基本语法如下：
```python
# 格式化整数
age = 25
print("我今年 %d 岁。" % age)

# 格式化浮点数
pi = 3.1415926
print("圆周率约为 %.2f。" % pi)

# 格式化字符串
name = "Alice"
print("你好，我的名字是 %s。" % name)
```
在上述代码中，`%d` 用于格式化整数，`%.2f` 用于格式化浮点数并保留两位小数，`%s` 用于格式化字符串。

### 新风格格式化（str.format()）
新风格格式化使用 `str.format()` 方法，它提供了更灵活和强大的格式化功能。语法如下：
```python
# 基本使用
age = 25
print("我今年 {} 岁。".format(age))

# 多个参数
name = "Bob"
print("你好，我的名字是 {}，我今年 {} 岁。".format(name, age))

# 通过索引指定参数位置
print("我叫 {1}，今年 {0} 岁。".format(age, name))

# 格式化数字
pi = 3.1415926
print("圆周率约为 {:.2f}。".format(pi))
```
`str.format()` 方法通过花括号 `{}` 来标记需要替换的位置，并且可以通过多种方式指定参数的位置和格式化方式。

### f - 字符串格式化（Python 3.6+）
f - 字符串格式化是 Python 3.6 引入的一种简洁而强大的格式化方式。它在字符串前加上 `f` 前缀，允许在花括号内直接嵌入表达式。
```python
age = 25
name = "Charlie"
print(f"我叫 {name}，今年 {age} 岁。")

# 格式化表达式
print(f"2 + 3 = {2 + 3}")

# 格式化数字
pi = 3.1415926
print(f"圆周率约为 {pi:.2f}。")
```
f - 字符串格式化使得代码更加简洁易读，并且在性能上也有一定的提升。

## 常见实践

### 格式化数字
- **指定小数位数**：
```python
number = 123.456789
print(f"{number:.2f}")  # 输出 123.46
```
- **添加千位分隔符**：
```python
large_number = 123456789
print(f"{large_number:,}")  # 输出 123,456,789
```

### 格式化字符串
- **填充和对齐**：
```python
text = "Python"
print(f"{text:^20}")  # 居中对齐，宽度为 20
print(f"{text:<20}")  # 左对齐，宽度为 20
print(f"{text:>20}")  # 右对齐，宽度为 20
```

### 日期和时间格式化
```python
import datetime

now = datetime.datetime.now()
print(f"{now:%Y-%m-%d %H:%M:%S}")  # 输出当前日期和时间，格式为 YYYY - MM - DD HH:MM:SS
```

## 最佳实践

### 保持一致性
在整个项目中，尽量使用一种格式化方式，以保持代码的一致性和可读性。如果项目中的代码风格倾向于简洁现代，f - 字符串格式化是一个不错的选择；如果需要兼容较旧的 Python 版本，则可以考虑使用 `str.format()` 方法。

### 提高可读性
格式化字符串应该清晰明了，避免过于复杂的表达式。如果需要进行复杂的计算或处理，可以先将结果存储在变量中，然后再进行格式化输出。

### 性能考虑
f - 字符串格式化在性能上通常优于旧风格格式化和 `str.format()` 方法。在性能敏感的代码段中，应优先使用 f - 字符串格式化。

## 小结
Python 打印格式化提供了多种方式来控制文本输出的外观，每种方式都有其特点和适用场景。旧风格格式化简单直接，适合快速编写代码；新风格格式化 `str.format()` 更加灵活强大，适用于复杂的格式化需求；f - 字符串格式化简洁高效，是 Python 3.6 及以上版本的首选。通过掌握这些方法，并遵循最佳实践，我们可以编写更加整洁、易读且高效的代码，提升程序的输出质量。
