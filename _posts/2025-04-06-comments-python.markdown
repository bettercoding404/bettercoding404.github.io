## 基础概念
Python 中的注释是代码中被解释器忽略的部分，其作用是为程序员提供关于代码的额外信息。注释可以帮助理解代码的功能、意图以及如何使用。在 Python 中，有两种主要的注释类型：单行注释和多行注释。

## 使用方法
### 单行注释
在 Python 中，单行注释以 `#` 符号开头，从 `#` 符号开始到该行末尾的所有内容都会被视为注释。
```python
# 这是一个单行注释
print("Hello, World!")  # 这里的注释解释了这行代码的作用
```

### 多行注释
虽然 Python 没有专门的语法来标识多行注释，但可以使用三个引号（单引号 `'''` 或双引号 `"""`）来实现类似效果。这种方式通常用于在代码中添加较长的注释块。
```python
'''
这是一个多行注释
可以跨越多行
用于提供更详细的解释
'''
def greet(name):
    """
    这个函数用于向指定的人打招呼
    :param name: 要打招呼的人的名字
    :return: 打招呼的字符串
    """
    return f"Hello, {name}!"
```

## 常见实践
### 解释代码功能
通过注释解释代码的具体功能，有助于其他开发者（或自己在未来）快速理解代码的意图。
```python
# 计算两个数的和
def add_numbers(a, b):
    return a + b
```

### 标记待办事项
在开发过程中，可以使用注释标记需要完成的任务。
```python
def some_function():
    # TODO: 实现这个函数的具体逻辑
    pass
```

### 文档化代码
使用特殊的注释方式为代码添加文档。例如，函数和类的文档字符串（docstring）。
```python
class Person:
    """
    Person 类用于表示一个人
    包含姓名和年龄属性
    """
    def __init__(self, name, age):
        """
        初始化 Person 实例
        :param name: 人的姓名
        :param age: 人的年龄
        """
        self.name = name
        self.age = age
```

## 最佳实践
### 保持简洁明了
注释应该简洁地表达核心内容，避免冗长复杂的表述。例如：
```python
# 计算圆的面积
def calculate_area(radius):
    return 3.14159 * radius ** 2
```

### 与代码逻辑同步更新
当代码逻辑发生变化时，及时更新相应的注释，以确保注释的准确性。

### 避免过度注释
不要对每一行代码都添加注释，只对关键部分、复杂逻辑或难以理解的代码添加注释。例如：
```python
# 计算阶乘
def factorial(n):
    result = 1
    for i in range(1, n + 1):  # 循环从 1 到 n
        result *= i
    return result
```

## 小结
Python 中的注释是提高代码可读性和可维护性的重要工具。通过合理使用单行注释和多行注释，以及遵循常见实践和最佳实践，开发者能够更好地编写易于理解和协作的代码。注释不仅有助于他人理解代码，也方便自己在日后回顾和修改代码时快速掌握代码的核心逻辑。
