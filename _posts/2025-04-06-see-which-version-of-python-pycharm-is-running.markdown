## 基础概念
PyCharm 是一款专为 Python 开发设计的集成开发环境（IDE），它支持使用不同版本的 Python 解释器。Python 解释器是执行 Python 代码的程序，不同的版本在功能、性能和兼容性上有所不同。例如，Python 2 和 Python 3 之间存在许多语法和库的差异。当我们在 PyCharm 中创建项目或运行代码时，需要指定一个 Python 解释器，而这个解释器的版本就是 PyCharm 运行的 Python 版本。

## 使用方法

### 在 PyCharm 界面中查看
1. **打开项目设置**：在 PyCharm 主界面，点击菜单栏中的 `File` -> `Settings`（Windows/Linux）或 `PyCharm` -> `Preferences`（Mac）。
2. **进入项目解释器设置**：在弹出的设置窗口中，找到并点击 `Project: [项目名称]` -> `Python Interpreter`。
3. **查看 Python 版本**：在 `Python Interpreter` 页面中，你可以看到当前项目所使用的 Python 解释器路径，在解释器路径旁边会显示对应的 Python 版本号。例如，`C:\Python39\python.exe (Python 3.9.1)`。

### 通过代码查看
在 Python 脚本中，我们可以使用 `sys` 模块来获取当前运行的 Python 版本信息。以下是示例代码：
```python
import sys

print(sys.version)
```
在 PyCharm 中运行上述代码，控制台会输出当前 Python 解释器的详细版本信息，例如：`3.9.1 (tags/v3.9.1:1e5d33e, Dec  7 2020, 17:08:21) [MSC v.1927 64 bit (AMD64)]`。

## 常见实践

### 项目迁移时查看版本
当我们将项目从一个开发环境迁移到另一个环境时，首先要查看原项目在 PyCharm 中运行的 Python 版本。这是因为不同版本的 Python 可能会导致代码出现兼容性问题。例如，如果原项目是在 Python 3.6 下开发的，而新环境默认的 Python 版本是 3.9，某些库的使用方式可能会发生变化，需要对代码进行相应调整。通过上述方法查看原项目的 Python 版本后，可以在新环境中安装相同版本的 Python 解释器，以确保项目能够顺利运行。

### 调试兼容性问题时查看版本
在调试过程中，如果遇到代码在某些功能上无法正常工作，或者出现奇怪的错误提示，有可能是因为 Python 版本的兼容性问题。此时，查看 PyCharm 运行的 Python 版本就显得尤为重要。通过确认版本信息，我们可以查阅相关的文档和资料，了解该版本与代码中使用的库或语法之间是否存在已知的兼容性问题。例如，某些旧版本的库可能在 Python 3.8 以上版本中需要进行更新或调整使用方式。

## 最佳实践

### 定期检查版本
随着项目的开发和维护，Python 解释器以及相关的库可能会发生更新。定期检查 PyCharm 运行的 Python 版本可以及时发现潜在的兼容性问题。建议在每次进行较大规模的代码更新或新功能开发前，查看一下当前的 Python 版本，并与项目文档中记录的版本进行对比。如果发现版本不一致，及时评估可能对项目产生的影响，并进行相应的处理。

### 记录项目使用的 Python 版本
为了方便后续的维护和协作，在项目文档中明确记录项目所使用的 Python 版本是一个很好的习惯。可以在项目的 README 文件或者专门的项目文档中，添加关于 Python 版本的信息。例如：
```
### Python 版本
本项目使用 Python 3.7.4 版本进行开发。请确保在运行项目时，使用相同或兼容的 Python 版本，以避免出现兼容性问题。
```
这样，新加入项目的开发人员可以快速了解项目的 Python 版本要求，减少因版本不一致导致的问题。

## 小结
了解 PyCharm 运行的 Python 版本对于 Python 开发者来说是一项重要的技能。通过在 PyCharm 界面中查看或使用代码查看版本信息，我们可以更好地应对项目迁移、调试兼容性问题等常见情况。遵循定期检查版本和记录项目使用的 Python 版本等最佳实践，能够提高项目的稳定性和可维护性。希望本文的内容能帮助读者更深入地理解并高效使用查看 PyCharm 运行的 Python 版本这一功能。
